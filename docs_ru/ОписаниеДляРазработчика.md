# Техническое описание проекта "StockUp" (Управление семейными покупками)

Документ предназначен для разработчиков. Здесь описаны основные архитектурные решения, стек технологий, схема данных и план реализации.

## 1. Архитектура и Стек технологий
- **Формат:** Telegram Web App (TMA).
- **Инфраструктура:** Монорепозиторий. Локальный запуск всех сервисов обеспечивается через `docker-compose`.
- **Основной API и Бот:** Python (FastAPI + Aiogram). Отвечает за CRUD-операции, обработку бизнес-логики и взаимодействие с Telegram.
- **База данных:** PostgreSQL. Используется единая база, изоляция данных между семьями (multi-tenancy) достигается за счет строгой фильтрации по `family_id`.
- **Микросервис уведомлений:** Golang. Легковесный воркер, который слушает брокер сообщений (Redis) и рассылает push-уведомления через Telegram API (асинхронно, не блокируя основной API).
- **Фронтенд:** SPA. Стилизация под нативные цвета мессенджера через CSS-переменные Telegram (например, `var(--tg-theme-bg-color)`).

## 2. Основная бизнес-логика
- **Авторизация:** Без логинов/паролей. Бэкенд валидирует криптографическую подпись строки `initData` от Telegram.
- **Семьи:** Один аккаунт = одна семья. Присоединение происходит по коду-приглашению (`/get_invite_code`). Вход в новую семью автоматически исключает пользователя из предыдущей.
- **Продукты и визуал:** 
  - В целях экономии места на сервере фотографии товаров не хранятся. 
  - Python-бэкенд использует `pymorphy3` для приведения названия товара к начальной форме (лемме), очистки от цифр и автоматического подбора **эмодзи** (иконки товара).
  - Эмодзи (иконка) и **категория** товара — это разные сущности. Эмодзи ставится автоматически (замена фото), а категория нужна для поиска/группировки (бэкенд может ее предложить, но пользователь вправе задать или изменить).
- **Уведомления:** 
  - Запросы на покупку отправляются всем членам семьи.
  - Подписка на продукты: при исчерпании количества (остаток = 0) подписанные пользователи получают автоматическое уведомление.

## 3. Схема данных (Сущности)

### ER-Модель
- `Families` (Семьи)
- `Users` (Пользователи)
- `Categories` (Категории продуктов внутри семьи)
- `Products` (Продукты)
- `Product_Subscriptions` (Подписки пользователей на уведомления по продуктам)

### Описание таблиц
1. **Families**
   - `id` (UUID)
   - `invite_code` (Уникальный код для входа)
   - `created_at`
2. **Users**
   - `telegram_id` (PK)
   - `family_id` (FK на Families)
   - `username`, `first_name`
3. **Categories**
   - `id` (UUID)
   - `family_id` (FK на Families)
   - `name` (Название категории, например "Молочные продукты")
4. **Products**
   - `id` (UUID)
   - `family_id` (FK на Families - изоляция)
   - `category_id` (FK на Categories)
   - `name` (Оригинальное название)
   - `lemma` (Нормализованное название для поиска)
   - `emoji` (Автоматически подобранная иконка)
   - `description` (Где купить, срок годности)
   - `quantity` (Остаток, может быть дробным)
   - `unit` (Единица измерения: кг, шт, л)
5. **Product_Subscriptions**
   - `user_id` (FK на Users)
   - `product_id` (FK на Products)

## 4. План реализации (Roadmap)
1. **Инициализация инфраструктуры:**
   - Настройка `docker-compose.yml` (Postgres, Redis, Python-бэкенд, Go-воркер).
   - Создание миграций для БД (таблицы описанные выше).
2. **Разработка Python Backend (FastAPI + Aiogram):**
   - Реализация проверки `initData` от Telegram.
   - CRUD для семей (генерация инвайт-кодов, вступление/выход).
   - Интеграция `pymorphy3` для парсинга названий и подбора эмодзи.
   - CRUD для продуктов и категорий (с обязательной фильтрацией по `family_id`).
   - Логика изменения количества товара и отправки событий в Redis при исчерпании остатка.
3. **Разработка Go Worker:**
   - Подключение к Redis, прослушивание очереди `notifications`.
   - Интеграция с Telegram Bot API для отправки сообщений (запросы на покупку и уведомления по подписке).
4. **Разработка Frontend (Web App):**
   - Создание UI с использованием CSS-переменных Telegram.
   - Реализация интерфейса добавления продукта (ввод названия, выбор категории, отображение авто-эмодзи).
   - Интерфейс списка продуктов с возможностью изменения количества.
